GitHub Pages is a static site hosting service that allows us to put our local webpages on the Internet. In order to use GitHub Pages, first you’ll need a GitHub account. GitHub is a website so we can just go to the www.github.com and create a login.

Here, keep your account public. It's the free option and just means that anyone can see your projects (which isn't a huge deal right now).

Next, we’ll need to navigate back to the homepage by clicking the cat icon in the top left corner. Once on the homepage, go to your email and click the link to verify so we can create a new project. Once your email is verified, go ahead and press the Start a Project button.

Now, we can set up our project (website) that will be on the Internet. Here, it says Create a Repository and a repository is just another name for project or folder. Don't get too caught up in the terminology for now - just know that this is where the code for our webpage is going to go.

Name your project your username.github.io. So if your username is harrypotter, your project name should be harrypotter.github.io. It is very important that it’s named this way because that’s how we get access to GitHub’s free hosting service (GitHub Pages). Go ahead and keep this project public - which just means anyone can see the code - and initialize it with a README. The README usually contains an in depth description of the project, which you can write later. Clicking submit...

And now we have our project! Now how do we get our website code there? Well, we can just create new files and code it all here online or you could upload files from your computer. Another option is to use the command line, which would allow us to code this locally on our own computer with Sublime and then upload it to GitHub through the text-based terminal. If you are interested in that, check out GitHub Terminal Pt. 2 video. You could also do this with the applications that GitHub has, but doing it from the terminal feels a lot cooler.

To use the terminal with git, you’ll need to download the git. The easiest way to do this is to download the applications provided by GitHub by clicking Clone or Download and then Open in Desktop. For the purposes of this tutorial, we will just be using the github website to write our code.

To start developing our website, we’ll go ahead and click Create New File and call it index.html. Inside of here, we’ll just create a simple header that says Hello World.

Then, we'll scroll down and commit the change by pressing Commit new file. In the text box, you can add a description about the change you made, but we'll just keep it blank for this tutorial.

Now, if the go to our github.io website, we’ll be able see the results of our code.

And there we see our Hello World. If we go to another computer and try to access this website, we'll also see Hello World. Right now, anyone in the world can go to a yourusername.github.io website and they will see the results of whatever code is written in the yourusername.github.io repository.

What if we want this header to say Hello Bob instead? Well, we can go back to GitHub and click the file.

Then, we can press the pencil icon and edit the file to have Hello Bob instead along with some extra text.

After commiting the change, we can refresh the website and now we see Hello Bob and I am a hacker. If you don't see this change immediately on your website, try waiting five minutes and refresh again.

You just learned how to use GitHub Pages!! Remember, this will only work with static sites so there can’t be any dynamic angular directives or routing going on because that won’t work. However, you can have directives and routing on other hosting sites, such as Heroku.
